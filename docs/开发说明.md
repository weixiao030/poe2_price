# 开发说明

本分支是 POE2 三服合一物价补丁，支持：

- 国服 WeGame Bundles2
- 国际服官方独立端 GGPK
- 国际服 Steam/Epic Bundles2

## 更新流程

1. 默认把 `物价补丁` 文件夹上一级当作游戏根目录，也可以用 `-Poe2Dir` 手动指定。
2. `Get-Poe2InstallInfo` 检测安装类型和目标语言资源。
3. 提取英文 `data/balance/baseitemtypes.datc64`。
4. 提取目标语言 `BaseItemTypes.datc64`：
   - 国服：`data/balance/simplified chinese/baseitemtypes.datc64`
   - 国际服：读取 `poe2_production*_Config.ini` 的 `language=` 后选择对应语言文件；例如繁中为 `data/balance/traditional chinese/baseitemtypes.datc64`，英文为 `data/balance/baseitemtypes.datc64`
5. 提取英文 `Words.datc64`、目标语言 `Words.datc64` 和 `UniqueGoldPrices.datc64`，用于识别传奇物品名并清理旧版传奇标价。
6. `build_poe2scout_price_patch.py` 按客户端类型抓取价格：国际服继续使用 poe2scout，国服 WeGame 优先使用 `https://poecurrency.top/api/summary?version=2`，没有国服数据的普通物品和传奇装备再用 poe2scout 价格兜底。普通物品写目标语言 `BaseItemTypes.datc64`。传奇物品默认写入兼容标记 `[价格|传奇名]`：PoE Overlay II 和易刷在解析 `Ctrl+C` 文本时都会把它清洗回原始传奇名，避免价格污染 `name` 查询字段，同时游戏内只显示一次传奇名。
7. D/E 换算比例实时从当前数据源推导：国际服读取 poe2scout，国服读取 poecurrency.top 中“神圣石”的当前价格；读不到比例时中止生成，避免使用固定比例。
8. 国服优先使用 `latest_buy1` / `latest_sell1` 取最新盘口价，缺失时再回退到 `buy_avg` / `sell_avg`：两边都有值且价差不超过 5 倍时取几何均值，价差超过 5 倍时取较低一侧，避免过期均价、OCR 异常或极端盘口把价格拉偏；只有单边有效时使用单边价格。
7. 按格式安装：
   - GGPK：`PatchBundledGGPK3.dll`
   - Bundles2：`PatchBundle3`

## 还原流程

Bundles2 模式更新前会生成 `真实还原物价补丁.zip`，备份当前 `Bundles2` 关键物理文件：`_.index.bin`、`_.index.high.bin`、`_.index.low.bin`、`.index.dbg` 和当时存在的 `LibGGPK3`。还原时优先使用这个物理备份包，把 Bundles2 恢复到安装本物价补丁前的状态。

普通 `还原物价补丁.zip` 仍会生成，用于记录干净 BaseItemTypes 和 Words 内容，但 Bundles2 真实还原不再用 PatchBundle3 写回这些资源来冒充还原。生成或刷新还原包时会拒绝带传奇价格标记的 `Words.datc64`，包括旧版独立行、Overlay 专用 `<<...>>` 和当前兼容标记。

GGPK 模式如果没有干净还原包，不从当前已修改 GGPK 反提取还原包，避免污染备份。

## 调试命令

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\物价补丁\tools\update_price_patch.ps1 -Poe2Dir "<POE2游戏根目录>" -NoInstall -NoPoe2dbFallback
```

## 构建

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\build\make_release.ps1
```

构建发布版前需要准备还原种子包，优先放在项目内 `restore-seeds/`，也兼容放在仓库父目录：

```text
restore-seeds/
├── 国服还原包.zip
└── 国际服还原补丁.zip
```

这些 zip 属于本地构建素材，不提交到 GitHub 仓库。
