# 开发说明

## 更新流程

1. C# 启动器解密内置 payload 到临时目录。
2. 启动器设置 `POE2_PATCH_ROOT` 和 `POE2_PATCH_RELEASE=1`。
3. PowerShell 脚本优先使用发布目录内置的 `tools/dotnet-runtime/dotnet.exe` 和 `tools/python/python.exe`。
4. `GGPKExtractor` 提取 `BaseItemTypes.datc64`。
5. Python 脚本抓取 poe2scout 价格，生成新的 `物价补丁.zip`。
6. `PatchBundledGGPK3.dll` 把补丁写入 `Content.ggpk`。

## 还原流程

还原脚本会使用 `还原物价补丁.zip`，通过同一个底层补丁工具把原版 `baseitemtypes.datc64` 写回游戏包。

## 发布版离线运行时

发布版运行时不会下载环境：

- 缺少 `tools/dotnet-runtime/dotnet.exe` 时直接报“发布包不完整”。
- 缺少 `tools/python/python.exe` 或 Python 依赖时直接报“发布包不完整”。

源码开发模式不设置 `POE2_PATCH_RELEASE=1` 时，可以继续使用系统 Python/.NET 或下载本地运行时。
