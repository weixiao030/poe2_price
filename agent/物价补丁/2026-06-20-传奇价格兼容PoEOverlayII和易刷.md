# 价格标记兼容 PoE Overlay II 和易刷

## 背景

旧版传奇标价格式 `传奇名\n[1.00E]` 会让 PoE Overlay II 的标题区变成三行，解析失败。此前的止血方案把默认模式改成 `off`，但这只是禁用传奇 Words 标价，不满足同时兼容的需求。

## 解析结论

- PoE Overlay II 会先把 `[A|B]` 清洗成 `B`，随后只对 `<<...>>` 做额外清理。
- 易刷会把 `[A|B]` 清洗成 `B`，并会额外删除单尖括号 `<...>`。
- `<<[1.00E]>>` 只兼容 Overlay：易刷会残留 `<>`。
- `<[1.00E]>` 只兼容易刷：Overlay 会残留 `<1.00E>`。
- `传奇名\n[1.00E]` 只兼容易刷：Overlay 标题区三行失败。

## 变更

- 默认 `--unique-price-label-mode` 改为 `markup`。
- `markup` 生成格式为 `[价格|传奇名]`，复制文本进入 Overlay / 易刷后都会还原为干净的 `传奇名`，并避免游戏内重复显示传奇名。
- 普通底材 `BaseItemTypes.datc64` 也生成 `[价格|底材名]`，避免稀有物品标题区出现单独价格行。
- 保留 `overlay` 和 `newline` 作为显式调试模式，但 `overlay` 不再作为默认，因为它不兼容易刷。
- `Test-WordsLookPatched` 和 Python 检测逻辑同步识别三类传奇价格标记：
  - `\n[1.00E]`
  - `<<[1.00E]>>`
  - `[1.00E|传奇名]`

## 注意

`[A|B]` 是两个查价器都支持的复制文本清洗格式，两个查价器都会保留 `B` 作为干净名称。`[传奇名|价格]` 会被清洗成价格而不是名称，因此不能作为兼容格式。
