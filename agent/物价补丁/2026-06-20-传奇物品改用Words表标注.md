# 传奇物品改用 Words 表标注

## 变更时间
2026-06-20

## 变更内容
- 传奇物品价格不再尝试匹配 `BaseItemTypes.datc64`，避免把 `Mageblood` 等独特名误认为底材名。
- 新增 `Words.datc64` 解析逻辑，按固定结构读取：
  - row size: `64`
  - 英文名字段：`row_start + 4`
  - 当前语言显示名字段：`row_start + 48`
- 新增 `UniqueGoldPrices.datc64` 解析逻辑，按固定结构读取：
  - row size: `20`
  - `row_start + 0` 为 `Words` row index
- 更新流程会提取英文 `words.datc64`、当前语言 `words.datc64` 和 `uniquegoldprices.datc64`。
- poe2scout 传奇价格会通过英文名匹配 `UniqueGoldPrices -> Words`，然后把价格追加到当前语言 `Words` 显示名字段。
- 生成的补丁 zip 会同时包含当前语言 `baseitemtypes.datc64` 和 `words.datc64`。
- Bundles2 安装时直接使用完整补丁 zip，避免只写入 BaseItemTypes 导致传奇价格不生效。
- 传奇物品 `Words` 标注范围限制为 English 和 Traditional Chinese；其他语言继续只标注 BaseItemTypes 中的普通物品/通货价格。
- poe2scout 传奇接口固定抓取 accessory、armour、weapon 三类，每类 `PerPage=100`。
- 还原逻辑同步支持 `words.datc64`：English / Traditional Chinese 的普通 restore zip 必须包含当前语言 `words.datc64`，避免还原后残留传奇物品价格。

## 变更原因
`BaseItemTypes.datc64` 只包含 `Utility Belt`、`Staff` 等底材名，不包含 `Mageblood`、`Headhunter` 这类传奇物品名。传奇物品名实际来自 `Words.datc64`，需要通过 `UniqueGoldPrices.datc64` 中的 `Name` 字段定位对应的 `Words` 行。
